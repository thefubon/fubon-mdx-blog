# Привет, мир!

Это моя первая статья в блоге, созданном на Next.js 15 с использованием MDX.

## Что такое MDX?

MDX позволяет использовать JSX в ваших Markdown документах. Вы можете импортировать компоненты, такие как интерактивные диаграммы или оповещения, и встраивать их в ваш контент.

```jsx
function Button() {
  return (
    <button onClick={() => alert('Нажата кнопка!')}>
      Нажми меня
    </button>
  )
}

<Button />
```

<Callout>
  MDX сочетает в себе мощь Markdown и возможности React.
</Callout>

<Callout type="warning" title="Внимание!">
  Обратите внимание, что вы должны настроить Next.js для работы с MDX.
</Callout>

<Callout type="success" title="Совет">
  Кастомные компоненты делают ваш блог более интерактивным и привлекательным.
</Callout>

<Callout type="error" title="Ошибка">
  Этот компонент не будет работать, если вы не зарегистрируете его в MDX провайдере.
</Callout>

## Аккордеон

<Accordion title="Что такое JSX?">
  JSX (JavaScript XML) — это расширение синтаксиса JavaScript, которое позволяет
  писать HTML-подобный код внутри JavaScript. Он используется в React для
  описания того, как UI должен выглядеть.
</Accordion>

<Accordion title="Зачем использовать TypeScript?">
  TypeScript добавляет статическую типизацию в JavaScript, что помогает
  обнаруживать ошибки на этапе компиляции, улучшает инструментарий разработки
  и делает код более поддерживаемым.
</Accordion>
